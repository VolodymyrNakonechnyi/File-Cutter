# Скорочувач TXT файлів

Це Python Qt застосунок з українським інтерфейсом для файлових операцій над `.txt` файлами. Основний сценарій: вибрати один TXT файл або папку, задати числовий цільовий розмір у `B`, `KB` або `MB`, і скоротити тільки ті TXT файли, які більші за цей розмір. За бажанням можна ввімкнути дозаповнення менших файлів мовоподібним випадковим текстом до цільового розміру.

## Вимоги

- Python 3.10 або новіший
- Графічне середовище з Qt-залежностями

## Встановлення на Ubuntu

```bash
cd /home/adminv/FileConfigurator
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e ".[dev]"
```

Якщо Qt не запускається через `xcb`, встановіть системні залежності:

```bash
sudo apt install -y libxcb-cursor0 libxkbcommon-x11-0 libxcb-xinerama0 libegl1
```

## Запуск

```bash
cd /home/adminv/FileConfigurator
source .venv/bin/activate
python -m file_configurator
```

Або без активації:

```bash
cd /home/adminv/FileConfigurator
.venv/bin/python -m file_configurator
```

## Перевірка

```bash
python -m pytest
python -c "from file_configurator.app import main; print(callable(main))"
```

## Мокові файли для тестування

Щоб створити багато тестових файлів, виконайте:

```bash
python scripts/create_mock_txt_files.py
```

Команда створює папку `mock_txt_files/` зі 110 файлами:

- 50 файлів приблизно по 100 KB.
- 50 файлів приблизно по 200 KB.
- 10 файлів приблизно по 10 KB для перевірки, що менші файли не змінюються.

## Як працює скорочення

- Можна вибрати один `.txt` файл або папку.
- У папці обробляються лише `.txt` файли верхнього рівня.
- Підпапки не обробляються.
- Файли інших типів пропускаються.
- За замовчуванням файли, які вже не перевищують цільовий розмір, не перезаписуються.
- Якщо ввімкнути опцію дозаповнення, менші файли доповнюються мовоподібним випадковим TXT контентом до цільового розміру.
- Мовоподібний filler генерується дуже маленькою вбудованою character-level моделлю без PyTorch/TensorFlow і без зовнішніх файлів моделі.
- Під час обробки показується прогрес-бар з кількістю оброблених файлів.
- Після завершення показується короткий підсумок без повного списку всіх файлів.
- Розмір задається тільки числом, а одиниця вибирається окремо: `B`, `KB`, `MB`.
- Скорочені файли перезаписуються на місці.
- Операція незворотна: автоматичних резервних копій немає.
- Текст лишається валідним UTF-8, тому фактичний розмір може бути трохи меншим за цільовий, якщо на межі був багатобайтовий символ.
